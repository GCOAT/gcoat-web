# Midjourney Image Guide — GCOAT Website

> All prompts are tuned to GCOAT's dark theme (`#0b1220` bg) with color variants for each brand scheme.
> Use Midjourney v6+ for best results.

## Brand Color Schemes — Quick Reference

Your site supports 5 switchable color schemes. Use these hex values in your prompts:

| Scheme | Primary Accent | Secondary Accent | Vibe |
|--------|---------------|-----------------|------|
| **Monochrome** (default) | `#6B7280` cool gray | `#4080CC` soft blue | Clean, neutral, professional |
| **Electric** | `#5500F0` deep violet | `#E0E000` chartreuse yellow | Bold, high-energy, saturated |
| **Midnight** | `#9933FF` purple | `#00C5CC` cyan | Moody, premium, mysterious |
| **Ember** | `#E01040` crimson red | `#F0A500` warm gold | Warm, urgent, powerful |
| **Forest** | `#20B860` emerald green | `#80C020` lime | Natural, growth, fresh |

All schemes share the same dark backgrounds: `#0b1220` (page), `#111b2e` (cards/surfaces).

> **Tip:** Generate backgrounds for your default scheme (Monochrome) first. Then for the other 4, just
> swap the accent color words in the prompt and re-run. Or generate one neutral version and tint it
> in Photoshop per scheme using Color Balance / Hue Shift.

### Step-by-Step: Background Photos

**Step 1: Set Up Midjourney**
- Use `/settings` in Discord and select the latest model (v6+)
- Turn on **High Variation Mode** for more diverse outputs
- Use **Remix Mode** (`/prefer remix`) so you can tweak prompts on upscales

**Step 2: Generate Initial Batches**
- Paste each prompt into `/imagine`
- You'll get 4 variations per prompt
- Re-roll 2–3 times per prompt before settling — you want options

**Step 3: Refine with Variations**
- Pick the best from each batch and hit **V1–V4** for close alternatives
- Use **Pan** (arrows) to extend the composition
- Use **Zoom Out** if it feels too tight

**Step 4: Upscale**
- Hit **U1–U4** to upscale the winner
- Then use **Upscale (2x)** or **Upscale (4x)** — you want at least 2560px wide

**Step 5: Color Correct Per Scheme**
- Open in Photoshop, Figma, or Photopea (free)
- Adjust **Color Balance / Hue/Saturation** so accent tones match the target scheme:
  - **Monochrome**: cool gray `#6B7280` + soft blue `#4080CC`
  - **Electric**: violet `#5500F0` + chartreuse `#E0E000`
  - **Midnight**: purple `#9933FF` + cyan `#00C5CC`
  - **Ember**: crimson `#E01040` + gold `#F0A500`
  - **Forest**: emerald `#20B860` + lime `#80C020`
- Darks should always match `#0b1220`
- Add a dark overlay (`#0b1220` at 40–60% opacity) for text readability
- **Shortcut**: Generate only the Monochrome version, then duplicate and shift hue in Photoshop (Image → Adjustments → Hue/Saturation) to create the other 4 variants

**Step 6: Optimize for Web**
- Export as **WebP** (preferred) or JPEG at 80% quality
- Create 3 sizes: full (2560px), tablet (1280px), mobile (768px) for `srcset`
- Target under 200KB per image
- Use `loading="lazy"` on all non-hero images

**Step 7: Implement with CSS Overlays**
```css
background: linear-gradient(to bottom, rgba(11,18,32,0.85), rgba(11,18,32,0.95)),
            url('your-image.webp');
```

### Pro Tips for Backgrounds
- **Style consistency**: After you love one image, use `--sref [image URL]` on all other prompts
- **Negative prompts**: Add `--no text, words, letters, people, faces, hands` to every prompt
- **Seed lock**: Note the seed (envelope reaction → check seed), reuse with `--seed [number]`
- **Aspect ratios**: `--ar 16:9` for standard, `--ar 21:9` for CTA band, `--ar 9:16` for mobile hero

## Part 2: Form & Chat UX Improvement Ideas

Your current `start.html` already has a strong dual-mode system (Chat vs Form toggle) with draft saving, progress bar, inline summary editing, and confetti success. Here are targeted improvements to make it more intuitive:

### Current Pain Points to Address

1. **Chat mode can feel slow** — typing indicator delays (400–900ms per message) add up across 7 steps
2. **No visual preview of what you're building** — users fill out fields but don't see how their project might look
3. **Feature selection (chips) lacks context** — "Hero", "About", "Services" mean nothing to non-dev clients
4. **Budget/Timeline step is combined** — two decisions crammed into one step feels rushed
5. **No back button in chat** — users can only edit via the summary at the end
6. **Form mode is a wall of fields** — no visual hierarchy beyond the two-column grid

### Recommended UX Improvements

#### A. Add Visual Examples to Feature Selection
When users pick features (step 4 in chat, chip group in form), show a small thumbnail or icon preview of what each feature looks like. This is where **example pics** come in — see Part 3.

#### B. Split Budget & Timeline Into Separate Steps
Currently step 7 is a combined substep. Splitting them gives each decision breathing room and keeps the conversational flow natural.

#### C. Add a "Back" Action in Chat Mode
Let users go back one step instead of waiting for the summary to edit. A subtle "← Go back" link below the input area.

#### D. Reduce Typing Delays
Drop the typing indicator to 200–400ms. The novelty wears off after step 2 — speed matters more.

#### E. Add Smart Defaults Based on Project Type
After the user picks "Landing Page" in step 3, pre-select common features (Hero, About, Contact Form) so they just confirm instead of picking from scratch.

#### F. Show a Live Preview Card
As users fill out the chat, build a small floating card on the side (desktop) or at the bottom (mobile) that shows their project summary updating in real time — name, type, features selected, budget.

#### G. Add Micro-Copy That Explains Fields
Instead of just "Hero" as a chip label, show "Hero — big intro banner at the top". One-line descriptions make non-technical clients confident in their choices.

### Step-by-Step: Example / Portfolio Screenshots

**Step 1: Generate the Base Images**
- Run each prompt through `/imagine`
- For project mockups (1–6), use `--ar 16:9`
- For feature previews (7–10), use `--ar 3:2` (tighter crop)

**Step 2: Pick the Most Realistic Ones**
- Midjourney sometimes makes UI that looks "almost right but off" — pick the ones with the cleanest layouts
- Avoid ones with garbled text — you'll replace text anyway

**Step 3: Clean Up in Figma or Photoshop**
- Remove any generated text (it'll be gibberish) and replace with clean placeholder text or leave blank
- Adjust colors to match your palette if needed
- For feature thumbnails: crop tightly to just the section

**Step 4: Add Browser Chrome (Optional)**
- For portfolio cards, wrap the screenshot in a browser frame mockup
- Free browser mockup templates are available on Figma Community
- This makes them look like real project screenshots

**Step 5: Create Multiple Sizes**
- **Portfolio cards**: 800×500px (2x for retina: 1600×1000)
- **Feature thumbnails**: 400×267px (2x: 800×534)
- **Service card backgrounds**: 600×400px

**Step 6: Optimize**
- Export as WebP, target 50–100KB per image
- Name them semantically: `portfolio-landing-page.webp`, `feature-preview-about.webp`

**Step 7: Alternative to Midjourney for Mockups**
If Midjourney's UI screenshots look too "AI-generated", consider these alternatives:
- **Screenshot real templates** from Dribbble, ThemeForest, or your own past work (with permission)
- **Use Figma** to design clean mockups yourself — more control, pixel-perfect
- **Use [shots.so](https://shots.so)** or **[brandbird.app](https://brandbird.app)** to wrap real screenshots in device frames
- Midjourney is best for the abstract/artistic backgrounds (Part 1), while real tool screenshots work better for UI examples

With Option B, use a CSS tint layer that responds to the active scheme:
```css
/* Example: tint background per active color scheme */
.section--hero::before {
  content: "";
  position: absolute;
  inset: 0;
  background: linear-gradient(
    135deg,
    hsla(var(--hue-primary), var(--sat-primary), var(--light-primary), 0.15),
    hsla(var(--hue-secondary), var(--sat-secondary), var(--light-secondary), 0.10)
  );
  mix-blend-mode: color;
  pointer-events: none;
}
```
This automatically tints any neutral background with the active scheme's accent colors.
